# The Helia 객실현황 평면도 UI/UX 설계 문서 (디자인 핸드오프)

## 1) 목적
- 기존 `객실 카드 클릭 -> 상세/수정/삭제(StayFormDrawer)` 기능은 유지한다.
- 객실 카드를 단순 그리드가 아닌 실제 동선/위치 기반 평면도 형태로 배치해 가시성을 높인다.
- 5F/6F 각각의 중앙 공용공간을 명확히 표현한다.
  - 5F: 신생아실 중심
  - 6F: 다용도실, 에스테틱, 스파 중심

---

## 2) 현재 동작 기준(유지 대상)
현행 코드 기준 아래 기능은 유지한다.

- 객실 카드 클릭 시 해당 객실의 산모/신생아/입퇴실 정보 확인 및 수정/삭제
- 객실 이동(드래그) 및 스왑 확인 다이얼로그
- 입실/퇴실/신생아/산모 통계 카드
- 필터(`전체`, `Prestige`, `VIP`, `VVIP`, `퇴실임박`)
- 과거데이터 탭

## 3) 디자인 방향

## 3.1 핵심 원칙
- 위치 인지 우선: 사용자가 객실 번호보다 “어느 동/라인/코어 주변인지”를 먼저 인식할 수 있어야 한다.
- 정보 밀도 균형: 평면도에서 카드 크기를 과도하게 키우지 않고, 핵심 상태(입실중/비어있음/퇴실임박/입실예정)를 1차 노출한다.
- 기존 업무 흐름 보존: 상세 편집은 기존 Sheet를 그대로 사용한다.

## 3.2 UI 톤
- 평면도 배경은 중립 회색/아이보리 계열
- 객실 상태색은 현재 시스템의 의미를 유지
  - 입실중: 기본 강조
  - 비어있음: 점선/중립
  - 퇴실임박: 노랑(D-2), 빨강(D-1/D-Day)
  - 입실예정: 블루 보조

## 5) 층별 평면 구성 가이드

아래는 디자이너가 레이아웃을 잡을 때 사용할 “구조 가이드”다.  
실측 치수 재현이 목적이 아니라, **상대적 위치와 동선 인지**가 목적이다.  
즉, 도면을 보고 “어림잡아 비슷한 위치”로 표현하면 충분하다.

## 5.1 5F (신생아실 중심)
- 상단 객실 라인 + 하단 객실 라인 + 중앙 코어 구조
- 중앙 영역에 `신생아실`을 시각적 중심 노드로 배치
- 계단/엘리베이터/복도는 중앙 좌측~중앙부에 코어로 표현
- 객실 노드는 실제 도면의 상대 순서(좌→우) 유지

권장 와이어(개념):

```text
┌──────────────────────────────────────────────┐
│ 상단 객실 라인 (5xx)                         │
│                 [사무실]  [501]   [502]      │
│                                              │
│                 [신생아실1]                  │
│                                              │
│                 [신생아실2]                  │
│                                              │
│ 하단 객실 라인 (5xx)                         │
│ [503][505][506]...[508]                      │
└──────────────────────────────────────────────┘
```

## 5.2 6F (공용 서비스 중심)
- 상단 객실 라인 + 하단 객실 라인 + 중앙 공용공간 구조
- 중앙 공용영역에 `다용도실`, `에스테틱`, `스파`를 분리된 블록으로 표현
- 코어(계단/엘리베이터) 위치는 5F와 유사한 인지 구조로 통일

권장 와이어(개념):

```text
┌──────────────────────────────────────────────┐
│ 상단 객실 라인 (6xx)                         │
│ [607][608][609][610][611]                    │
│                                              │
│         [다용도실][에스테틱][스파]           │
│                                              │
│ 하단 객실 라인 (6xx)                         │
│ [601][602][603][605][606]                    │
└──────────────────────────────────────────────┘
```

---

## 5.3 객실 타입/번호 확정 규칙 (요구사항 반영)
- `Prestige`: `501`, `502`
- `VVIP`: `607` ~ `611`
- 그 외 객실: 모두 `VIP`
- 결번: `504`, `604`는 객실 슬롯/데이터/라벨에서 제외
- 객실 카드(노드) 크기는 **모든 방 동일 크기**로 통일

디자인 반영 원칙:
- 평면도 슬롯에는 실제 존재 객실만 배치한다.
- 결번은 빈 슬롯처럼 보이지 않게 하고, 복도/여백/시설 영역으로 처리한다.
- 타입 배지 색상은 `Prestige / VIP / VVIP`를 즉시 구분 가능하게 유지한다.
- 방 크기는 동일하게 유지하고, 위치만 도면 인상에 맞춰 대략 조정한다.

## 6) 객실 카드(노드) UX 스펙

## 6.1 카드 크기 계층
- 기본: `M` 사이즈 (객실 번호 + 상태 + 산모명)
- 축약: `S` 사이즈 (협소 구간용, 핵심만 표기)
- 확장(hover/focus): 퇴실일/입실예정 1줄 추가

## 6.2 카드 정보 우선순위
1. 객실 번호
2. 상태 배지(입실중/비어있음/퇴실임박)
3. 산모명
4. 보조정보(신생아 수, 입실예정, 퇴실일)

## 6.3 인터랙션
- `Click`: 기존 `StayFormDrawer` 오픈 (유지)
- `Drag`(데스크톱): 객실 이동/스왑 (기존 정책 유지)
- `Keyboard`: 포커스 이동 + Enter로 상세 오픈

---

## 7) 공용공간 노드 스펙

5F:
- `신생아실` 노드를 가장 높은 시각 우선순위로 표시
- 객실 카드보다 덜 강한 경계선으로 “영역” 느낌 강조

6F:
- `다용도실`, `에스테틱`, `스파` 3개 노드 분리
- 각 노드는 색조를 통일하되 아이콘/라벨로 구분

공통:
- 공용공간 노드는 기본적으로 편집 액션 없음
- 툴팁 또는 정보 팝오버만 허용

## 8) 반응형 전략

## 8.1 Desktop (주 사용 시나리오)
- 평면도 풀 레이아웃 + 카드 드래그 이동
- 통계 카드/범례 동시 노출

## 8.2 Tablet
- 동일 평면도 구조 유지
- 카드 밀집 시 자동 축약(S 카드)

## 8.3 Mobile
- **리스트 우선**으로 표시하고, 평면도는 2차(접기/펼치기 또는 보조 탭)로 제공
- 손가락 드래그로 객실 이동은 비활성(현행 정책 유지)
- 탭 시 상세 시트 오픈은 동일

---

## 9) 상태/색상 규칙(디자인 시스템 반영)

- `Occupied`: 기본 강조 톤
- `Empty`: 중립/점선
- `Checkout D-2`: 옐로 경고
- `Checkout D-1`, `D-Day`: 레드 경고
- `Upcoming`: 블루 보조 라벨

주의:
- 색상만으로 상태 전달 금지
- 아이콘 + 라벨 + 테두리 형태를 함께 사용
