2. UI/UX 구현 전략
① 뷰 전환 토글 (View Switcher)
상단 우측에 Lucide-React 같은 라이브러리의 아이콘(LayoutGrid, Calendar)을 활용해 탭 형태로 두 화면을 빠르게 오갈 수 있게 만듭니다.

② 엑셀 뷰 (Editable Data Grid)
컴포넌트: TanStack Table이나 react-datasheet 같은 라이브러리를 사용해 엑셀과 유사한 사용자 경험을 제공합니다.

기능: 방향키로 셀 이동, 대량 복사/붙여넣기, 클릭 시 드롭다운(D, E, N, /)으로 근무 변경이 가능하게 설계합니다.

③ 캘린더 뷰 (Interactive Event Calendar)
컴포넌트: FullCalendar 혹은 직접 구현한 커스텀 그리드를 사용합니다.

기능: 엑셀 뷰에서 입력된 데이터가 '이벤트 카드' 형태로 표시됩니다. 캘린더 내에서도 드래그 앤 드롭으로 근무 날짜를 옮기면 DB에 즉시 반영됩니다.

2.1 'The Helia' 대시보드 시나리오 예시
조리원 운영 시스템(가칭 'The Helia')의 관리자가 업무를 처리하는 흐름입니다.

대량 편성 (엑셀 뷰): 월초에 모든 간호사의 한 달 스케줄을 엑셀 뷰에서 빠르게 타이핑하거나 기존 데이터를 복사해 채웁니다. 이때 하단 합계 로직이 작동해 특정 인원의 근무가 너무 많지 않은지 체크합니다.

현황 파악 (캘린더 뷰): 운영 중 특정 날짜에 입실 산모가 갑자기 몰리는 것을 캘린더 뷰에서 확인합니다. (입실 숫자가 빨간색으로 강조됨)

유연한 수정 (공통): 캘린더 뷰에서 인원이 부족한 날을 확인하고, 그 자리에서 바로 다른 날의 근무자를 클릭해 근무 타입을 변경합니다. 이 수정사항은 엑셀 뷰에도 즉시 업데이트되어 나중에 정산할 때 정확한 수치를 보여줍니다.

3. UI/UX 상세 설계 가이드
3.1 실시간 객실 현황판 (Room Dashboard)
비주얼 시스템: 엑셀의 조건부 서식을 웹 UI 배지로 진화시킴.

퇴실 하루 전: 카드 테두리 Pink (#FFB6C1) 강조 및 "퇴실 D-1" 국문 배지 노출.

입실 하루 전: 카드 테두리 Blue (#ADD8E6) 강조 및 "입실 예정" 국문 배지 노출.

상태 표시: 산모명 옆에 신생아 수 아이콘(👶 x N) 배치. 특이사항(알러지 등)은 ⚠️ 아이콘 툴팁 처리.

3.2 스마트 근무표 (Smart Scheduler)
인력 충족도 헤더 (Live Indicator): * 화면 상단에 실시간으로 (현재 총 케어 가능 인원) / (현재 총 신생아 수)를 비교하여 표시.

가용량이 부족할 경우(Ratio < 1.0) 전체 대시보드에 "인력 충원 필요" 경고 알림 발생.

듀얼 뷰 토글:

엑셀 모드: TanStack Table 사용. 방향키 및 단축키(D, E, N)로 셀 데이터 즉시 수정.

캘린더 모드: FullCalendar 스타일. 일자별 근무자 명단을 직종별로 그룹화하여 표시.

5. AI 바이브 코딩 지시서 (AI Master Prompt)
상황: 너는 시니어 풀스택 개발자이며, 3년 차 React 개발자인 나의 파트너야. 미션: THE_HELIA_MASTER_SPEC.md의 설계를 바탕으로 Next.js 앱의 핵심 구조를 빌드해줘.

핵심 구현 순서:

DB Setup: Supabase SQL을 사용하여 staff, rooms, stays, schedules 테이블을 생성하고, 직종별 max_capacity를 설정하는 DB 트리거를 작성해.

API Logic: 특정 날짜의 stays 합계(신생아 수)와 schedules의 합계(가용 인원)를 비교하는 서버측 로직을 작성해.

UI - Room Card: shadcn/ui Card를 커스텀하여 엑셀 시트의 색상 규칙이 적용된 '객실 현황 카드' 컴포넌트를 만들어.

UI - Scheduler: TanStack Table을 활용해 엑셀처럼 빠르게 근무를 입력할 수 있는 그리드 뷰를 구현해.

Placement Algo: '파트타이머 우선 배치' 로직을 적용한 autoAssign Server Action을 작성해.
